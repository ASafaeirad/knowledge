LZ77 is a **dictionary-based lossless compression algorithm** developed by **Abraham Lempel and Jacob Ziv in 1977**. It's a pattern compression.
It compresses data by replacing repeated sequences with references to earlier occurrences.

## Core Idea
Instead of repeating data, LZ77 outputs a **reference**.

### Reference format:
`(distance, length, next_symbol)`
### Example:
```
Original:
ABABCABABC

Compressed idea:
ABABC (5,5) // Repeat the previous 5 characters from distance 5.
```
## Sliding Window
LZ77 uses a **sliding window** consisting of:
- **Search buffer** (past data)
- **Lookahead buffer** (future data)
## Key Properties
- Uses previously seen data as a dictionary
- Efficient for repetitive data
- Works in streaming environments

## Important Notes
- LZ77 alone does **not perform [[entropy coding]]**.
- It outputs many tokens, which are later compressed further.
- Algorithms like **[[DEFLATE]]** combine LZ77 with Huffman coding.
- Window size affects compression efficiency.
- Large windows detect more patterns but use more memory.
